## Installation Issues

### `npm install` fails with peer dependency errors

**Symptom:** Errors like `ERESOLVE unable to resolve dependency tree`

**Solution:**
```bash
npm install --legacy-peer-deps
```

### `bun install` fails

**Symptom:** Bun version compatibility error

**Solution:** Use npm instead:
```bash
npm install
```

## Development Server

### Port 5173 already in use

**Symptom:** `Error: listen EADDRINUSE: address already in use :::5173`

**Solution:** Kill the process using the port or specify a different port:
```bash
# Find and kill the process
lsof -ti:5173 | xargs kill -9

# Or start on a different port
npm run dev -- --port 3000
```

### Environment variables not loading

**Symptom:** `import.meta.env.VITE_*` returns `undefined`

**Solution:**
1. Ensure your file is named `.env` (not `.env.local` unless intentional)
2. All variables exposed to the browser **must** be prefixed with `VITE_`
3. Restart the dev server after changing `.env` files

## Build Errors

### `vite build` fails with out-of-memory error

**Symptom:** `JavaScript heap out of memory`

**Solution:**
```bash
NODE_OPTIONS=--max-old-space-size=4096 npm run build
```

### Module not found

**Symptom:** `Cannot find module '@/components/...'`

**Solution:** Ensure path aliases are configured in both `vite.config.ts` and `tsconfig.app.json`:
```ts
// vite.config.ts
resolve: {
  alias: { "@": path.resolve(__dirname, "./src") }
}
```
```json
// tsconfig.app.json
"paths": { "@/*": ["./src/*"] }
```

## TypeScript Errors

### Type errors in shadcn/ui components

**Symptom:** Red squiggles in `src/components/ui/` files

**Solution:** These files are generated by shadcn/ui. Re-generate the component:
```bash
npx shadcn@latest add <component-name>
```

### `Property does not exist on type 'ImportMeta'`

**Symptom:** TypeScript doesn't recognize `import.meta.env`

**Solution:** Ensure `src/vite-env.d.ts` exists and contains:
```ts
/// <reference types="vite/client" />
```

## Testing Issues

### Tests fail with `Cannot find module`

**Symptom:** Vitest cannot resolve path aliases during test runs

**Solution:** Ensure `vitest.config.ts` mirrors the Vite path alias configuration:
```ts
import { defineConfig } from 'vitest/config';
import path from 'path';

export default defineConfig({
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') },
  },
  test: {
    environment: 'jsdom',
  },
});
```

### `ReferenceError: document is not defined`

**Symptom:** Test crashes because browser APIs are unavailable

**Solution:** Set the test environment to `jsdom` in `vitest.config.ts`:
```ts
test: {
  environment: 'jsdom',
  setupFiles: ['./src/test/setup.ts'],
}
```

## Deployment

### Vercel build fails

**Symptom:** Build succeeds locally but fails on Vercel

**Checklist:**
1. Ensure all required `VITE_*` environment variables are set in the Vercel project settings
2. Verify the **Build Command** is `npm run build` and **Output Directory** is `dist`
3. Check the Node.js version in Vercel matches your local version (`node --version`)

### Blank page after deploy

**Symptom:** The app loads but shows a white/blank page

**Solution:** If you are using React Router with client-side routing, add a `vercel.json` rewrite rule:
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```
